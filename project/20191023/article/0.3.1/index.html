Gist Note
created at 2019/09/19
俺はリフィルを世界中の学生に作ってもらいたい。雑誌のDTPとか。かわいいスタンプとか。そういったものだ。技術者にも良い意味でデザインが必要だ。俺はデザイナとデベロパの手を結ばせるのだ。ひいてはデザイナとデベロパに使ってもらいたいツールだ。このアプリを開きGISTHUBにログインしGISTを作れば即座にウェブ上に情報を公開できる。勿論シークレットも可能だ。SEMANTIC-UIの強力なタグ機能はアメリカ人が新しいものを次々作り出すのと同じ勢いでアジア人もポイントを夜空の星星のように紡ぐことが出来る。そして作り出した記事は強力なリフィル機能によって纏められ、GITHUB上の世界中のプラグインによってスライドショーや画像のビュアーなどを組み込める。数式やシンタックスハイライタもだ。リフィルとして登録する時に記事中本文にuuidとかあると、プラグイン（スクリプト）が処理するのに必要な個別のUUIDがテンプレートエンジンによって割り当てられて登録される。 俺は世界を変えられる。なぜなら世界中で利用されている最高のサービスの上にアプリを作るからだ。強力な地盤。インフラの整った街に高層ビルを建設する。もはやGISTはそれ単体で使われなくなる。誰もが俺のURLを介して利用されるだろう。そのとき、俺は本当の意味で広大なネットワークを手にする。 だが、仕掛けが必要だ。世界中の開発者や表現者、この世界に新しい息吹をもたらし技術によって物事の味方を買える者たちが好きな時に、好きな用に世界中の開発者のライブラリを自由に使って自己を表現しようとする時、このGIST NOTEは役立つ。強力なリフィル機能は表現を豊かにする。 welcome to gist note, gist.jsとrepo.jsをメインとして世界中の開発者に好まれるエディタを作る。ガラケー時代のfreespaceみたいに誰でも簡単にリフィルで書ける gist.js -> login -> create gist する。http://ohtruss.github.io/note/gist-hash俺のURLを解すので全く心配ない。人は自由になりたがる。敢えて選択肢の狭める（タグの数）ことで考える力の大きい企業ほど金を払うようになる ファイルホスティングの時代は終わった。これからはバックエンド側おユーザ数とか登録とかタグ数に制限をかける 市場を操作して付加価値をあげなければ。金をかける基準は量から室に変わり次に種類の数が来る
